Answer Relevancy on 19 goldens — judged by gpt-5-mini

A live run of DeepEval's AnswerRelevancyMetric against the ShopSphere chatbot, with every golden scored by the judge LLM and the result documented case by case. The same run is shown side by side against the earlier gpt-4o-mini judge so you can see exactly what the judge upgrade changed.

AWhat's being tested, and who's judging

Two different LLMs play two different roles. The model under test generates the answers; the judge LLM grades them. They are never the same model — that separation is the whole point of LLM-as-judge.

BResult & the judge upgrade, side by side

Same chatbot, same 19 goldens, same threshold (0.7). Only the judge model changed. The newer judge reverses six false failures.

DCase-by-case detail

Each golden: the question, the golden's expected answer, the chatbot's live reply, then both judges' score + reason for that reply.

EHow to read the failures

The metric is referenceless. AnswerRelevancyMetric scores only how on-topic the reply is for the question — it never reads the golden expected_output. So a "fail" here means "the reply added something the question didn't ask for," not "the reply is wrong." To grade against the golden answer, use a correctness G-Eval metric that consumes expected_output.
Why gpt-5-mini scores higher. The remaining 4 fails are genuine verbosity (the bot bundles an extra sentence — free-shipping note, customs note, price on a "specs" question). The six cases that flipped were correct answers the older gpt-4o-mini judge mis-scored to 0.0–0.5 with self-contradicting reasons. The judge upgrade fixed judge noise, not the chatbot.
